Reflection on proposal

Compared with the previous mockup, the design of some details and the location of some information have been improved. Iterated through user feedback. Some operation logic has been improved. For the details of the previous proposal,click here.

Iteration

I did two iterations and the feedback each time helped me improve my website experience. The first iteration focused more on the overall design of the website, and the second iteration paid more attention to the details of the website.

First Iteration

In the first iteration, the overall design style of the website was improved. After getting user feedback, the navigation bar was first optimized. The previous version did not do a good job of distinguishing the current page. Therefore, using a more concise design can let users know the current page better.

Added more information quick navigation in the home page, users can see more information titles. If users are interested in a certain piece of information, they can click on the quick link to go to a specific website for details.

By increasing the number of pictures, users can see more pictures at once. The previous design only had a single photo, and it was difficult for users to see more photos on the home page. Therefore, remove the gallery page in the final prototype and choose to increase the number of pictures on the home page to achieve the function of the gallery page.

Adding the introduction of recent activities at the bottom of the page can help users know the latest development of the foundation.

More introductory information has been added to the about us page to facilitate users to better understand Flight Safety Foundation. At the same time, by adding the video function, users can have another choice to learn about the foundation.

Second Iteration

Based on the first iteration, I did the second iteration. The second iteration user feedback focused on the Service and Donate pages. And users are satisfied with the results of the first iteration.

In the service interface, users feel that the content is too redundant and not concise. Users need to see more content summaries to determine whether to make a deeper understanding of the content of interest. Therefore, adding different types of information helps users find content that interests them more quickly.

In the donate page, users are confused about donate-related issues, so I added a FAQ section. The number and frequency of donate have also been improved. Corresponding explanations are also made for the expenses incurred in payment.

Final Prototype

After two iterations, users are satisfied with the effect of the final prototype.
